# 🌎🏆 World Cup 2026 Pool — Live Dashboard

A live, auto-updating World Cup 2026 pool for you and 15 friends. Deployed on GitHub Pages for free. Scores auto-sync every hour from [football-data.org](https://www.football-data.org/).

## Deploy in ~15 minutes

### Step 1 — Create a new GitHub repository

1. Go to [github.com/new](https://github.com/new)
2. Name it something like `worldcup-pool` (can be anything)
3. Keep it **Public** (required for free GitHub Pages + free GitHub Actions)
4. Check **"Add a README file"**
5. Click **Create repository**

### Step 2 — Upload these files

You have two options:

**Option A — Web upload (easiest, no git needed):**
1. In your new repo, click **Add file → Upload files**
2. Drag *every file and folder* from this project into the upload box. Make sure the `.github/workflows/sync.yml` file ends up at that exact path — web upload usually preserves folders if you drag the folder itself.
3. Scroll down, click **Commit changes**

**Option B — Using git (if you're comfortable):**
```bash
git clone https://github.com/YOUR_USERNAME/worldcup-pool.git
cd worldcup-pool
# copy all files from this project into the folder, then:
git add .
git commit -m "Initial setup"
git push
```

### Step 3 — Add your football-data.org API key as a secret

1. In your repo, go to **Settings** (top nav)
2. In the left sidebar, click **Secrets and variables → Actions**
3. Click **New repository secret**
4. **Name:** `FOOTBALL_DATA_TOKEN` (must be exact)
5. **Secret:** paste your football-data.org API token
6. Click **Add secret**

### Step 4 — Enable GitHub Pages

1. In your repo, go to **Settings → Pages** (left sidebar)
2. Under **Source**, pick **Deploy from a branch**
3. **Branch:** `main` · **Folder:** `/ (root)` · click **Save**
4. Wait ~1 minute. Your live URL will appear at the top of the Pages page. Looks like:
   `https://YOUR_USERNAME.github.io/worldcup-pool/`

Share that URL with your friends.

### Step 5 — Run your first sync

1. Go to the **Actions** tab in your repo
2. Click the **Sync World Cup scores** workflow on the left
3. Click **Run workflow** → **Run workflow** (green button)
4. Wait ~30 seconds. When it turns green, your `stats.json` has been updated.

After this, it runs automatically every hour.

Before the tournament starts (June 11, 2026), the sync will write "no matches yet" — that's fine. Once the first matches happen, the leaderboard will start moving on its own.

## How it works

```
┌────────────────────────────────────────────────────┐
│ GitHub Pages serves index.html at your URL         │
│   └── index.html fetches config.json, draw.json,   │
│       stats.json on every page load                │
└────────────────────────────────────────────────────┘
            ▲
            │ (every 2 min, browsers re-fetch stats)
            │
┌───────────┴────────────────────────────────────────┐
│ GitHub Actions (sync.yml) runs every hour          │
│   1. Fetches WC 2026 matches from football-data.org│
│   2. Computes team stats per your scoring rules    │
│   3. Commits updated stats.json back to the repo   │
└────────────────────────────────────────────────────┘
```

## Changing things after launch

### Edit player names or team lists

1. Open `data/config.json` in GitHub's web editor (click the file, then the pencil ✏️)
2. Edit, commit
3. If you changed pots or player lists, regenerate the draw next.

### Re-run the draw

You need Node.js 20+ locally:
```bash
git pull
npm run draw
git add data/draw.json
git commit -m "new draw"
git push
```

Or skip Node and manually edit `data/draw.json` via GitHub's web editor.

### Manually override a team's stats

If the auto-sync gets something wrong, you can override specific fields. Edit `data/stats.json` via GitHub's web editor and fill in the `manualOverrides` object:

```json
{
  "manualOverrides": {
    "Brazil": { "wonR32": true, "cleanSheets": 2 },
    "Argentina": { "groupFinish": 1, "advancedKO": true }
  }
}
```

On the next sync, these overrides get reapplied *after* the auto-fetch, so they persist.

### Change the sync frequency

Edit `.github/workflows/sync.yml`, find the cron line:
```yaml
- cron: "5 * * * *"   # every hour at :05
```
Use [crontab.guru](https://crontab.guru/) to pick a different schedule. Free-tier football-data.org allows 10 requests/minute, so hourly is safe.

## Troubleshooting

**Leaderboard shows all zeros and says "no matches":**
The tournament hasn't started, or fixtures haven't been published in football-data.org yet. Normal — it'll auto-fill once matches are played.

**Sync status says "error":**
Go to the **Actions** tab, click the latest failed run, and read the logs. Common issues:
- Token not set correctly → check Step 3
- API rate limit → wait, re-run manually
- football-data.org doesn't have this competition in free tier → see "Alternative data sources" below

**A team's name isn't matching between the API and your config:**
Add an entry to `apiFootballNameAliases` in `data/config.json`. Example:
```json
"apiFootballNameAliases": {
  "USA": ["United States", "United States of America"],
  "Türkiye": ["Turkey", "Turkiye"]
}
```

**Superstar goals aren't showing up:**
Player name matching is approximate. If the API spells a name differently (accents, transliteration), edit `data/config.json` `pot2[].superstar` to match how football-data.org spells it.

## Alternative data sources

If football-data.org doesn't have what you need, the sync script is easy to re-point:

| Source | Notes |
|---|---|
| **football-data.org** (default) | Free. Email signup only. Good WC coverage. |
| **API-Football (via RapidAPI)** | Free 100/day. More detailed events. Requires RapidAPI account. |
| **TheSportsDB** | No signup. Free. Less reliable for live data. |
| **ESPN hidden endpoints** | No signup. Unofficial. Can break. |

To switch sources, modify `scripts/sync.mjs`. The rest of the app reads from `data/stats.json` in a fixed schema, so any source that can write to that schema works.
